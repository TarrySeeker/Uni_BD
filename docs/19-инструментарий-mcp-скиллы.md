# Инструментарий: MCP-серверы, скиллы, агенты (admik)

> Единая политика авто-применения MCP-серверов, скиллов и привязки инструментов к субагентам.
> Краткая выжимка — в `CLAUDE.md` (блок «Инструментарий»). Здесь — подробно.
> Принцип: MCP и скиллы — часть обязательного флоу (две проверки, TDD, сверка документации), а не «по желанию».

---

## 0. Механика Claude Code (по официальной документации)

Источники: `code.claude.com/docs/en/subagents`, `.../mcp-servers`, `.../skills`, `.../hooks`.

- Поле `tools` во frontmatter субагента — **comma-separated allowlist**. Если поле **опущено** — агент наследует ВСЕ инструменты (built-in + все MCP). Если **указано** — это исключающий список: при добавлении MCP нужно перечислить заново и старые tools, и новые MCP-записи.
- Синтаксис MCP в `tools`:
  - **Весь сервер:** `mcp__<server>` либо `mcp__<server>__*` (эквивалентны).
  - **Поштучно:** `mcp__<server>__<tool>` (точные имена доступны только при подключённом сервере).
  - Wildcard `mcp__*` работает в `disallowedTools` (запрет всех MCP), в `tools` — нет.
- Пока playwright/context7/postgres в статусе ⏸/заблокированы — используем форму «весь сервер» (`mcp__<server>`): не зависит от точных имён тулзов. Добавлять `mcp__<server>` в allowlist ДО подключения сервера безопасно — тулзы просто неактивны, включатся автоматически после старта сервера.
- **Авто-применение** обеспечивается ИНСТРУКЦИЯМИ (этот документ + блок в `CLAUDE.md` + промпты агентов), которые загружаются в контекст; hooks для тяжёлых проверок НЕ используются (см. §5).

## 1. MCP-серверы

### 1.1 playwright — браузерная автоматизация
**Статус:** ⏸ ждёт одобрения (project scope в `.mcp.json`; браузеры в `~/.cache/ms-playwright`; без ключа).
**Одобрение:** запустить `claude` в проекте → подтвердить диалог доверия серверов из `.mcp.json` (или `/mcp` → Approve).

**Авто-триггеры (ОБЯЗАН):**
- После ЛЮБОЙ правки UI витрины/админки — «ручная проверка» из правила двух проверок.
- При закрытии задачи с визуальным/интерактивным результатом: форма, кнопка, статус заказа, корзина, оплата, фильтр каталога.
- После деплоя на стенд — smoke ключевых путей на реальном URL.

**Примеры задач admik:** оформление заказа каталог→подтверждение; подписка на рассылку (G-12); форма обратной связи (G-09); demo-страница Т-Банка в mock и возврат на success/fail; фильтры каталога и фото из MinIO; смена статуса заказа отражается на витрине (G-15); один сценарий на admik и autoparts (нет хардкода магазина).

**Не применять / предостережения:** не замена Vitest для логики; не гонять по проду с реальной оплатой/СДЭК — только mock; не вводить реальные секреты; флаку изолировать, не «чинить» отключением проверки.

### 1.2 context7 — актуальная документация библиотек
**Статус:** ⏸ ждёт одобрения (project scope; без обязательного ключа).

**Авто-триггеры (ДОЛЖЕН):**
- Перед использованием API свежих зависимостей: Next.js 16 (App Router, Server Actions), React 19, Zod 4, postgres.js, ioredis, TanStack Table, Tiptap, React Hook Form, @dnd-kit.
- При написании теста ДО кода (TDD), если контракт библиотеки неочевиден — сверить до фиксации ожидания.
- При ошибке типов/рантайма, похожей на смену API между версиями.

**Не применять / предостережения:** не для бизнес-логики admik (домен — в `docs/` и памяти); не подменять чтение нашего кода; примеры с хардкодом/секретами адаптировать под мультитенантность и env.

### 1.3 firecrawl — web-скрейпинг/краулинг
**Статус:** ✔ подключён (local scope; ключ в env вне репо). ⚠️ пакет требует Node ≥22 (у нас Node 20 — работает; при странных ошибках апгрейдить Node). Ключ при случае ротировать.

**Авто-триггеры (ДОЛЖЕН):**
- Чтение онлайн-документации интеграций: API СДЭК (расчёт, ПВЗ, создание/трекинг отправлений, статусы, печать) и Т-Банк (оплата card/SBP, формат уведомлений).
- Импорт данных каталога: товары, характеристики, фото-источники.

**Не применять / предостережения:** не краулить большие сайты широким захватом — точечные страницы; ключ только из env, не в логи; скрейп товаров — мультитенантно (в конфиг/данные магазина); уважать robots/ToS.

### 1.4 glyph — символьные outline кода (tree-sitter)
**Статус:** ✔ подключён (project scope). Собран без sudo через `zig cc` (CGO; компилятора в системе не было). Бинарь — `~/.local/bin/glyph`; Go 1.26 — `~/go-sdk`; zig — `~/zig`. Пересборка: `CC="$HOME/zig/zig cc" CGO_ENABLED=1 GOBIN=$HOME/.local/bin go install github.com/benmyles/glyph@latest`.

**Авто-триггеры (ДОЛЖЕН):** навигация по большому TS-репо перед правкой (outline вместо чтения целиком); поиск точек расширения при добавлении модуля; трассировка паттерна мутаций по символам.
**Предостережения:** outline не заменяет чтение критичного участка; источник истины по API — context7, по поведению — playwright/тесты.

### 1.5 postgres — read-only MCP к БД
**Статус:** ✔ подключён (local scope) к **dev-БД** на `localhost:5433` (embedded-postgres 18.4, без sudo; полная схема — 31 миграция, 36 таблиц; роль `admik_readonly`). Пакет — `@modelcontextprotocol/server-postgres` (read-only by design).
**dev-БД эфемерна** (останавливается при перезагрузке) — поднять заново: `~/admik-devdb/start.sh` (вне репо). Роли: `admik_app`/`admik_migrator`/`admik_readonly`.
**Альтернатива (реальные данные):** read-only роль + SSH-туннель к стенду — по разрешению владельца.

**Авто-триггеры (после активации):** отладка реальных данных при баге заказа/каталога; сверка схемы перед миграцией; подтверждение, что мутация записалась и `audit_log` заполнен.
**Предостережения:** ТОЛЬКО read-only — запись лишь через Server Actions; креды из env; не тащить персональные данные клиентов в логи/контекст.

## 2. Скиллы (slash-команды)

### 2.1 Встроенные — когда применять автоматически
| Скилл | Когда автоматически в admik | Правило |
|---|---|---|
| **verify** / **run** | После правки бизнес-логики/UI — вторая из двух проверок (реальный прогон). | две проверки |
| **code-review** | Перед коммитом на рабочий diff; effort `high`+ для денег/RBAC/оплаты. | качество diff |
| **security-review** | Перед коммитом в путях оплаты, RBAC, токенов заказа, webhook СДЭК/Т-Банк, `.env`/секретов. | секреты из env |
| **deep-research** | Сверка актуального API интеграций, когда context7 недоступен (СДЭК/Т-Банк, миграции Next16/React19/zod4). | интеграции |
| **init** | Заведение нового магазина-инстанса — генерация/обновление CLAUDE.md инстанса. | мультитенантность |

Ситуативные (`simplify`, `fewer-permission-prompts`, `loop`, `schedule`, `update-config`, `review`, `claude-api`) в авто-флоу не вшиты.

### 2.2 Кастомные проектные скиллы
Расположение: `.claude/skills/<name>/SKILL.md`. ⚠️ = требует доступов стенда (память `admik-stand-deploy`, вне репо).

| Скилл | Назначение | ⚠️ |
|---|---|---|
| **gate** | Полный код-гейт перед коммитом (typecheck/lint/vitest/миграции/security). | |
| **deploy-stand** | Деплой на стенд + health-gate + откат (живая проверка). | ⚠️ |
| **new-shop** | Онбординг нового магазина-инстанса (мультитенантность). | |
| **audit-cycle** | Один цикл адверсариального аудита (owner-gated, кампания сошлась). | |
| **integration-check** | СДЭК+Т-Банк e2e в mock-режиме. | |

## 4. Порядок одобрения / активации

**MCP scope:** `--scope local` (приватно этому пользователю в проекте) · `--scope project` (`.mcp.json`, checked in) · `--scope user` (`~/.claude.json`, глобально).

- **playwright / context7** (project, ⏸ Pending approval): запустить `claude` в каталоге → одобрить серверы из `.mcp.json`; `claude mcp list` покажет статус.
- **firecrawl** (✔ local): уже работает; ключ в `~/.claude.json` (вне репо).
- **glyph** (нет gcc): см. §1.4 — поставить компилятор, собрать, добавить.
- **postgres** (нет БД): см. §1.5 — поднять БД/туннель, добавить с read-only строкой.

`.mcp.json` (playwright, context7) безопасен для коммита — секретов нет. firecrawl и postgres держим в local scope с кредами из env.

## 5. Hooks — почему НЕ внедряем
Hooks — детерминированные действия харнесса вне модели. Авто-`PostToolUse: Edit|Write → pnpm test/lint` для admik вреден: длинный vitest-сьют блокирует каждый Edit, flaky-e2e сорвут поток. Правило admik — гейт перед **коммитом** (скилл `gate`) + CI, а не после каждого Edit. Деплой/прямой SQL на стенд owner-gated — авто-hook здесь опасен (риск `rsync --delete` без `.env`). Поэтому авто-применение держим на инструкциях (CLAUDE.md + промпты агентов + этот документ), а не на hooks.

## 6. Инвариант
MCP и скиллы дополняют, но не отменяют правила admik: TDD (сначала тест), две проверки (код + браузер), мультитенантность (ноль хардкода под один магазин), секреты только из env, документирование каждой правки.
