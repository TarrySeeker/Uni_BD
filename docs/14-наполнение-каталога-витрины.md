# 14 — Наполнение каталога для витрины THE CASE (инструкция владельцу)

> **Автор:** Documentation Manager. **Статус:** инструкция для не-разработчика.
> **Дата:** 2026-06-16. **Связано:** `docs/13-сращивание-the-case.md` (контракт сращивания),
> `docs/05` (модель каталога), `docs/09` (разворачивание).
>
> **Назначение.** Пошагово показать, что и как завести в **админке Admik**, чтобы товар
> корректно и полно отобразился на витрине **THE CASE**. Витрина — это «обёртка»: она только
> показывает то, что есть в Admik. Своего каталога у неё нет.

---

## 0. Главный принцип

Витрина читает каталог из Admik через публичный API. Поэтому:
**завели/изменили товар в админке → он появился/обновился на витрине** (без участия программиста).
Витрине отдаются только товары со статусом **«Активен»**.

---

## 1. Категории (раздел «Каталог → Категории»)

1. Создайте дерево разделов витрины. Для THE CASE это обычно: верхний уровень
   **Женское / Мужское**, внутри — **Костюмы / Халаты / Аксессуары** (можно как удобно).
2. У каждой категории задайте **Название** и **slug** (латиницей, напр. `women`, `kostyumy`).
   Slug — это адрес раздела на витрине (`/catalog?category=<slug>`).
3. Категория должна быть **активна**, иначе она и её товары не покажутся.

> Навигация и вкладки каталога на витрине строятся **автоматически** из этого дерева —
> ничего хардкодить в коде витрины не нужно.

---

## 2. Товар (раздел «Каталог → Товары → Новый»)

Заполните карточку товара:

| Поле | Что писать | Где видно на витрине |
|---|---|---|
| **Название** | Название товара | заголовок карточки |
| **Slug** | латиницей (адрес товара `/product/<slug>`) | адрес страницы |
| **Статус** | **Активен** (иначе товар скрыт) | — |
| **Базовая цена** | цена в рублях | цена |
| **Старая цена** (compare-at) | если есть скидка — «цена было» | зачёркнутая цена + бейдж скидки |
| **Новинка** (is_new) | для бейджа «New» | бейдж New |
| **Рекомендуемый/Хит** (is_featured) | помечает «бестселлер» | блок Bestsellers, фильтр |
| **Бренд** | если используете бренды | бренд на карточке |
| **Категории** | привяжите к нужным разделам (можно к нескольким) | в каких разделах виден товар |

---

## 3. Характеристики товара (секция «Характеристики» в карточке товара)

Поля карточки THE CASE (состав/уход/цвет/особенности/пол) приходят **из атрибутов товара**.
Чтобы они заполнились — в системе должны существовать **атрибуты с такими кодами** (раздел
управления атрибутами), а в карточке товара им заданы значения:

| Поле на витрине | Код атрибута (важно!) | Пример значения |
|---|---|---|
| Пол | **`gender`** | `women` / `men` / `unisex` (или «женский/мужской/унисекс») |
| Цвет | **`color`** | `Графит` |
| Состав | **`composition`** | `70% хлопок, 25% полиэстер, 5% эластан` |
| Уход | **`care`** | `Стирка при 30°C, не отбеливать` |
| Особенности | **`features`** | `Воротник-стойка; Скрытые карманы; Боковые разрезы` |

Важно:
- **Код атрибута должен совпадать** (`gender`, `color`, `composition`, `care`, `features`) —
  по нему витрина и находит значение. Регистр и кириллица-дубли (`пол`, `цвет`, …) тоже
  распознаются, но канон — латинские коды выше.
- **Особенности** можно вводить как список или одной строкой через `;` / `,` — витрина разобьёт
  на пункты.
- Любой атрибут **необязателен**: если не задан — поле на витрине просто пустое, ошибки не будет.
- Пол, если не задан атрибутом, витрина пытается вывести из категории (женская/мужская).

---

## 4. Размеры = варианты (секция «Варианты» в карточке товара)

Каждый размер — это **отдельный вариант** товара. Так у каждого размера свой остаток и (при
желании) своя цена. Для каждого размера добавьте вариант:

| Поле варианта | Что писать |
|---|---|
| **Артикул (SKU)** | уникальный артикул (напр. `KE-001-M`) |
| **Название** | **метка размера** — напр. `M` (именно это покажет витрина как размер) |
| Цена (override) | заполняйте, только если цена этого размера отличается от базовой |
| Надбавка | альтернатива override — прибавка к базовой цене |

Важно:
- **Метка размера на витрине берётся из поля «Название» варианта.** Пишите туда сам размер
  (`XS`, `S`, `M`, `L`, `XL`, `XXL`). Размеры на карточке сортируются по канону `XS<S<M<L<XL<XXL`.
- Вариант должен быть **активен**.
- Без вариантов товар покажется как **«Нет в наличии»** (нечего класть в корзину).

> Техническая причина, почему размер пишется в «Название», а не в отдельный атрибут варианта —
> см. `docs/13 §3.3` (вариантные атрибуты пока не пробрасываются на витрину; запланировано
> улучшение).

---

## 5. Остатки (секция «Склад/Остатки»)

Задайте **количество** по каждому варианту (размеру). Витрина показывает только булево
«в наличии»: если остаток варианта > 0 — размер доступен для заказа; иначе — размер неактивен.
Точные числа остатков покупателю не раскрываются.

---

## 6. Изображения (секция «Медиа»)

1. Загрузите фотографии товара.
2. Отметьте одно фото как **главное** (is_primary) — оно идёт первым в галерее и на карточке
   в каталоге. Остальные показываются в галерее по порядку.

> Технлокальное: на стороне витрины разрешён домен медиа Admik (`NEXT_PUBLIC_ADMIK_API_URL`),
> поэтому картинки из админки отображаются на витрине без доп. настройки.

---

## 7. Проверка (после сохранения)

1. Товар со статусом «Активен» и хотя бы одним активным вариантом с остатком > 0.
2. Откройте витрину: товар виден в каталоге и в нужном разделе (категория), бейджи New/скидка —
   если заданы.
3. Карточка товара: цена, описание, состав/уход/цвет/особенности (если заполнены атрибуты),
   кнопки размеров (= варианты), фото.
4. Добавьте в корзину, перейдите к оформлению: расчёт доставки СДЭК (город → ПВЗ → стоимость),
   создание заказа. Заказ появится в админке Admik (раздел «Заказы»).

---

## 8. Минимальный чек-лист «товар готов к показу»

- [ ] Категория создана и активна, у товара проставлены категории.
- [ ] Товар: Название, Slug, **Статус = Активен**, Базовая цена.
- [ ] (опц.) Атрибуты `gender/color/composition/care/features`.
- [ ] Варианты-размеры: у каждого SKU + **Название = размер**, активен, остаток > 0.
- [ ] Хотя бы одно фото, одно — главное.
- [ ] Проверено на витрине: каталог → карточка → корзина → оформление.
