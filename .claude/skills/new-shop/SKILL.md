# Новый магазин-инстанс admik

Принцип (ADR-003): один магазин = один сервер = одна БД.
Код НЕ меняется под магазин. Перенос = ТОЛЬКО конфигурация (`.env`) + сид-данные.
НОЛЬ правок бизнес-логики и существующих миграций. Оригинал платформы не трогать.
Образец готового инстанса — отдельный магазин-копия (см. память `autoparts-instance-and-kit`).

## Шаги
1. **Скопировать платформу** в новый каталог: свежий git, ветка `main`,
   первый коммит = «импорт из admik@<hash>», второй = «ретематизация».
2. **Настроить `.env`** (из `.env.example`): `SHOP_NAME`, `ORDER_PREFIX`, `SHOP_DOMAIN`/`ADMIN_DOMAIN`,
   `NEXT_PUBLIC_ADMIK_API_URL`, `STOREFRONT_ALLOWED_ORIGINS`, `ADMIK_MODULES`
   (`catalog,orders,cdek,cms,payments`), `SEED_DEMO_CATALOG`. Секреты — только в `.env`, не в репо.
3. **Сид-данные** (если своя тематика): `db/seed/demo-catalog.sql` (категории + EAV-атрибуты + товары)
   и идемпотентный seed брендинга под флаг в `scripts/init-shop.sh`. Обнови соответствующий seed-тест.
4. **Документация:** README инстанса, `docs/00` (журнал), отдельный doc о переносе.
5. **Инициализация локально:** `make up` → `make init` (= `init-shop.sh`: миграции → seed прав/ролей/владельца →
   опц. демо-каталог) → `make smoke`.

## Боевой деплой инстанса (не-разработчик)
По `docs/09-инструкция-разворачивания.md`: bootstrap сервера, `cp .env.example .env` (домены,
пароли БД/MinIO, `STOREFRONT_API_KEY(S)`, `ORDER_TOKEN_SECRET`), `make deploy`.
На бою: `SEED_DEMO_CATALOG=false`, реальные секреты; ключи СДЭК/Т-Банк — только при включении модулей.

## Гейт приёмки инстанса
Прогони скилл `gate`: vitest 0 failed, `tsc` exit 0, `check-migrations` (аддитивные), `security-review` 0 находок.
Витрину подключай через integration-kit (см. память `autoparts-instance-and-kit`).

## Итог
Путь инстанса, изменённые конфиг/сид-файлы, результат гейта. Подключение витрины — по разрешению владельца.
